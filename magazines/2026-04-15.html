Supply Chain Security

Someone Bought 30 WordPress Plugins and Planted a Backdoor in Every One

An unknown threat actor acquired 30 legitimate WordPress plugins from their original developers, then pushed malicious updates to every site running them. Over 30,000 active installations received the backdoor automatically through WordPress's built-in update mechanism.

This is the supply chain attack pattern that keeps CISOs awake. The plugins were established, trusted, and reviewed. The new owner simply pushed an update containing obfuscated code that phones home to an external server and creates a hidden admin account. Because WordPress auto-updates plugins by default, most site owners had no idea anything changed.

The attack exploits a gap no technical control fully covers: the trust chain between a plugin marketplace, original authors, and the organisations that depend on them. For any organisation running WordPress (even for marketing microsites), this demands an immediate audit of installed plugins and their current ownership.

What to do: Audit all WordPress plugin ownership across your estate. Review auto-update policies. Consider tools that alert on plugin ownership changes. Add plugin supply chain risk to your third-party risk register.

Stop Flock: The Campaign Against Warrantless Mass Surveillance

Flock Safety's AI-powered licence plate cameras are now deployed at over 100,000 locations across the US, creating a searchable nationwide database of vehicle movements accessible to police without a warrant. Two HN front-page stories today push back hard.

The system goes far beyond reading plates. Flock's "Vehicle Fingerprint" technology identifies cars by colour, make, damage, bumper stickers and wheel type. Its "Convoy Analysis" feature detects vehicles that frequently travel near each other, mapping social associations without anyone's knowledge or consent.

A Kansas police chief used the system 228 times to stalk an ex-girlfriend. In Oak Park, Illinois, 84% of drivers stopped via Flock alerts were Black in a town that is 21% Black. A federal judge ruled the network functioned as a "dragnet over the entire city" and equated it to warrantless GPS tracking.

The Stop Flock campaign and a companion post from a privacy researcher who formally opted out of Flock's data collection both gained significant traction. This is the surveillance infrastructure debate that every democracy is going to have to settle.

Backblaze Silently Stops Backing Up OneDrive and Dropbox Folders

Backblaze has quietly stopped including OneDrive and Dropbox sync folders in its backup scope. Users are discovering the gap only when they need to restore. No announcement. No notification. Just silent data loss.

The original report from a Backblaze customer documents what happened: files stored in OneDrive and Dropbox directories simply stopped being backed up. The Backblaze interface continued to show "all files backed up" with no warnings or errors. The change appears to have been made server-side without user consent.

For organisations that layer Backblaze as a secondary backup behind cloud sync, this breaks assumptions about data resilience. The 3-2-1 backup rule only works if you verify all three copies actually exist.

What to do: Verify your backup scope today. Don't trust status dashboards. Run a test restore of files from cloud sync directories. If you're in a CISO role, add backup scope verification to your operational resilience checks.

OpenSSL 4.0.0 Released

The first major version bump since OpenSSL 3.0 in 2021. Version 4.0.0 brings a significant API and ABI break, updated algorithm defaults, and removes long-deprecated functions. Every team running OpenSSL needs to plan their migration.

This is not a drop-in upgrade. The release includes breaking changes to the provider architecture, removes the legacy provider by default, and changes several default algorithm parameters. Applications compiled against OpenSSL 3.x will need recompilation and likely code changes.

The good news: the security defaults are stronger out of the box. The bad news: any custom integrations, embedded devices, or legacy applications will need testing. For a CISO, this means coordinating with engineering teams to plan a migration timeline before the 3.x branch reaches end-of-life.

What to do: Inventory all OpenSSL dependencies across your estate. Start compatibility testing in non-production environments. Track the 3.x end-of-life date and build a migration plan.

Fiverr Left Customer Files Public and Searchable

A Hacker News user discovered that Fiverr's file storage allowed public access to customer deliverables, project files, and communications. The files were indexed and searchable through the platform's own URLs with no authentication required.

The disclosure (posted as a "Tell HN") details how changing parameters in file URLs exposed other customers' data. Deliverables from freelancers, client briefs, source files, and private messages were all accessible. The vulnerability appears to stem from predictable or enumerable file identifiers combined with missing access controls.

This is a textbook IDOR (Insecure Direct Object Reference) vulnerability at scale. Fiverr handles work for millions of businesses, including design assets, code, marketing materials, and sensitive internal documents. The exposure window and scope of affected data remain unclear.

Google's Gemma 4 model can now run natively on iPhone hardware with full offline inference. No network connection needed, no data leaving the device. This is the privacy-preserving AI deployment model that security-conscious organisations have been waiting for.
